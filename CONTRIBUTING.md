# Contributing to Curated Local AI

Thank you for your interest in contributing! This guide will help you add new tools or improve existing entries.

## How to Add a New Tool

1. **Fork** this repository
2. **Edit** `README.md` and add your tool to the appropriate category
3. **Submit** a pull request

### Entry Format

Each entry should follow this format:

```markdown
| [Tool Name](https://github.com/owner/repo) | Brief description (one sentence). | Platform/Backend | License |
```

### Requirements

- **Real projects only** -- the tool must exist and be publicly available
- **Working URL** -- all links must be valid and point to the official source
- **Accurate description** -- keep it factual and concise (one sentence)
- **Correct category** -- place the tool in the most relevant section
- **Alphabetical order** is preferred within each category but not strictly required
- **No duplicates** -- check if the tool is already listed
- **Open-source preferred** -- proprietary tools are accepted if they have a free tier

### What We Accept

- Inference engines and runtimes for local AI
- Desktop and web UIs for interacting with local models
- Quantization tools and methods
- Fine-tuning frameworks and tools
- Local RAG (Retrieval-Augmented Generation) solutions
- Voice and speech tools (STT, TTS)
- Local image generation tools
- Hardware guides and benchmarks for local AI

### What We Do Not Accept

- Cloud-only services with no local/self-hosted option
- Abandoned projects (no commits in 12+ months with unresolved issues)
- Tools that require paid API keys with no local alternative
- Promotional or spam submissions
- Duplicate entries

## Reporting Issues

If you find a broken link or incorrect information:

1. Open an [issue](https://github.com/awesome-ai-tools/curated-local-ai/issues/new)
2. Include the tool name and what needs to be fixed
3. Provide the correct information if possible

## Code of Conduct

Be respectful and constructive. We are all here because we share an interest in local AI.

## Questions?

Open an issue with the "question" label and we will do our best to help.
